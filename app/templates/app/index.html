WHAT'S DIFFERENT?

What is so special about B.R.I.E.?

Dissection of Genre

Forget the days where each book is given a specific genre. You select a book, and Brie tells you how much of romance, crime, fiction, et al are present in it. Select some more, voila! Your profile is generated.

A Hybrid Recommendation System

A recommendation system that views each book as a real world entity. Chucks out bias, disregards bestsellers and recommends books tailor made for your profile.

Deep Drilled Analytics

Are you an avid reader? A publisher? An author? No worries! B.R.I.E. has a detailed set of reports and dashboards to fulfill everyone's need in the publishing industry.

TECH PIPELINE

How was B.R.I.E. made?

  • Data Collection

    Web Crawlers and APIs

    Multiple websites such as goodreads, readgeek, wikipedia, amazon, etc were scraped. APIs were used in cases where they were available. Believe us, this was the most tedious part!

  • Data Cleaning and Integration

    Text Cleaning - MySQL and MongoDB

    B.R.I.E handles a large volume of text data (review, comments, description, etc), all of which are cleaned using different mechanisms such as stop words removal and lemmatization. The data is modelled carefully and based on the use cases encounterd, spit into two storages - MySQL and MongoDB.

  • Machine Learning

    Text Classification - Multinomial Text Models

    Variations of text classification models are used to diessect the genres of each book. A multinomial text model with multiple dependencies to bring out a human touch is implemented to come up with accurate book recommendations

  • UI/UX and Reports

    Django - Bootstrap - Plotly

    The django web framework is used to host the entire product. The frontend is written on HTML, javascript and CSS with bootstrap implementation. The reports and dynamically generated using Plotly

  • Voila!
    B.R.I.E is up
    and running
